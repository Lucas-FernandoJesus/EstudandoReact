## Recomendações não aplicadas

### Restaurar o ambiente padrão do projeto

- Problema: PowerShell 7.6.4 consta no perfil observado, mas `pwsh.exe` não está acessível; o caminho principal da skill não funciona no terminal atual.
- Proposta: executar conscientemente `setup\install.cmd` e depois verificar `pwsh -NoLogo -NoProfile -Command "$PSVersionTable.PSVersion"` em um novo terminal.
- Arquivos ou ferramenta afetados: instalação local/PATH do PowerShell e ferramentas gerenciadas pelo bootstrap; nenhum arquivo pedagógico deve ser alterado.
- Benefício esperado: retomadas e scripts passam a usar o shell padrão documentado, sem o fallback para Windows PowerShell.
- Risco ou efeito colateral: o bootstrap pode verificar ou atualizar componentes locais; deve ser iniciado pelo usuário e revisado antes da execução.
- Mudança de comportamento: os comandos documentados com `pwsh` voltarão a executar diretamente.

### Verificar a política do terminal após restaurar o PowerShell 7

- Problema: o shell hospedeiro bloqueou scripts, enquanto o processo Windows PowerShell separado apresentou `CurrentUser = RemoteSigned`.
- Proposta: abrir um terminal novo após o bootstrap e comparar `Get-ExecutionPolicy -List`; só investigar configuração adicional se o script continuar bloqueado.
- Arquivos ou ferramenta afetados: configuração de execução do PowerShell; nenhuma alteração é proposta antecipadamente.
- Benefício esperado: eliminar a necessidade do bypass por processo e esclarecer a diferença entre os ambientes.
- Risco ou efeito colateral: mudar a política sem diagnóstico pode ampliar a execução permitida; por isso a recomendação inicial é somente verificar.
- Mudança de comportamento: nenhuma até que uma causa seja confirmada e uma alteração específica seja autorizada.
