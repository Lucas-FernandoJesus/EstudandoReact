# Módulo 00 — Ambientação

## Objetivo

Compreender o ambiente de trabalho antes de escrever código: arquivos, caminhos, terminal, editor, navegador, versões e autorização para mudanças.

## Como usar este material

Este módulo contém conceitos e enunciados reutilizáveis. O ambiente atual e as preferências ficam em `learning/state/perfil.json`; respostas, tentativas e observações históricas ficam em `learning/logs/`.

Antes de ensinar uma ferramenta, verificar o ambiente atual e pedir uma explicação ou microtarefa que demonstre o conhecimento do estudante. A presença de uma ferramenta não comprova familiaridade com ela.

## Conceitos deste módulo

### Arquivo e pasta

Uma pasta organiza itens; um arquivo guarda conteúdo interpretado por uma ferramenta. A extensão ajuda o sistema e o editor a reconhecer o tipo, mas faz parte do nome do arquivo e não transforma conteúdo incorreto em código válido.

### Caminho

Um caminho absoluto começa em uma localização completa do sistema. Um caminho relativo descreve onde algo está a partir da pasta atual. O terminal sempre possui uma pasta de trabalho atual.

### Terminal e comando

O terminal é uma interface textual para conversar com programas. Um comando possui um programa, possíveis opções e argumentos. Antes de executar, é importante distinguir comandos de leitura de comandos que modificam o sistema.

### Editor e navegador

O editor ajuda a escrever e compreender arquivos. O navegador interpreta HTML, CSS e JavaScript e oferece DevTools para inspecionar a página, mensagens, rede e armazenamento.

### Versão

Ferramentas mudam. Registrar a versão torna instruções reproduzíveis e ajuda a identificar incompatibilidades. Uma versão instalada não significa automaticamente que ela seja a mais apropriada para todo projeto.

## Sequência sugerida

1. Aplicar apenas as partes necessárias do diagnóstico em `exercicios.md`.
2. Praticar identificação de caminhos e comandos não destrutivos.
3. Explicar o papel de editor, terminal, runtime, gerenciador de pacotes e Git.
4. Definir objetivo, rotina e primeiro módulo técnico.

Nenhum comando de instalação, configuração ou criação de projeto faz parte desta etapa sem autorização separada.
