# Trilha curricular

Usar esta ordem como mapa adaptável, não como calendário rígido. Diagnosticar antes de pular etapas.

## 00 — Ambientação

Arquivos e pastas, caminhos, terminal, editor, navegador, DevTools, instalação segura, versões, Git básico e leitura de mensagens de erro.

Critério: navegar no projeto, executar verificações não destrutivas e explicar a função das ferramentas principais.

## 01 — Fundamentos da Web

Cliente e servidor, HTTP em nível conceitual, URL, navegador, página estática, separação entre estrutura, estilo e comportamento.

## 02 — HTML

Documento, semântica, texto, links, imagens, listas, tabelas, formulários, validação nativa e acessibilidade.

## 03 — CSS

Cascata, especificidade, box model, unidades, tipografia, cores, Flexbox, Grid, responsividade, estados e organização de estilos.

## 04 — JavaScript

Valores, variáveis, operadores, condicionais, repetições, funções, arrays, objetos, escopo, DOM, eventos, módulos, JSON, Fetch, promessas, `async`/`await`, erros e depuração.

## 05 — Git e práticas de desenvolvimento

Repositório, estado dos arquivos, commits, branches, diffs, histórico, colaboração, documentação, terminal e DevTools. Introduzir Git cedo e aprofundar de forma transversal.

## 06 — TypeScript

Tipos básicos, inferência, unions, narrowing, objetos, funções, interfaces, aliases, generics, módulos, configuração e integração gradual com JavaScript.

Pré-requisito: JavaScript funcional, especialmente funções, objetos e módulos.

## 07 — React

Componentes, JSX, props, estado, eventos, renderização condicional, listas, formulários, composição, hooks essenciais, efeitos, compartilhamento de estado, acessibilidade, roteamento, testes e consumo de APIs.

## 08 — Tailwind CSS

Utility-first, responsividade, estados, composição, tema e integração com React. Ensinar depois de CSS para evitar memorização sem modelo mental.

## 09 — Node.js e APIs

Runtime, npm, módulos, eventos, assincronismo, HTTP, variáveis de ambiente, arquitetura de API, rotas, validação, erros, logs e testes. Introduzir framework somente após uma API mínima com fundamentos compreendidos.

## 10 — Dados e bancos

Modelagem, SQL, tabelas, chaves, relacionamentos, consultas, joins, agregações, transações, migrations, PostgreSQL e integração segura com backend.

## 11 — Segurança, autenticação e qualidade

Hash de senhas, sessão ou token, autorização, validação, CORS, segredos, OWASP em nível apropriado, testes unitários e de integração, lint, formatação e observabilidade básica.

## 12 — Full-stack e entrega

Integração frontend/backend, contratos de API, estados de carregamento e erro, deploy, CI básico, variáveis por ambiente, performance e projeto final incremental.

## Estrutura de cada módulo

1. Diagnóstico dos pré-requisitos.
2. Objetivos e vocabulário mínimo.
3. Demonstrações análogas.
4. Microexercícios.
5. Exercício integrador sem solução incluída.
6. Revisão por evidências.
7. Recuperação espaçada.
8. Critério de avanço e próximos passos.
